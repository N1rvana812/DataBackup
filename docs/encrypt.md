语境目标	系统管理员希望在备份时节省存储空间并保证数据机密性。
前置条件	1. 基础备份流程已启动。
2. 用户通过 --compress 和 --encrypt 开启功能，并提供了加密密码。
成功的结束状态	归档文件体积显著减小，且内容被RC4加密，无法被直接读取。
失败的结束状态	若压缩/加密底层库调用失败，中止备份并清理临时文件。
参与者	System Admin(系统管理员)
触发器	命令行传入 --compress --encrypt --password <pwd> 参数。
基础用例	UC-01 (执行基础备份)
包含用例	密钥派生 (Key Derivation)
扩展用例	无
事件流	基本流程	1. 系统读取4KB数据块。
		2. 将数据块送入 RLE 管道进行流式压缩。
		3. 压缩后的数据块送入 RC4 管道进行加密。
		4. 将密文写入磁盘归档文件。
		5. 循环直至文件读取完毕。
	扩展流程	无
非功能需求	严禁将整个文件加载到内存；密码不得以明文形式在内存中长期驻留。
补充规格说明书	压缩和加密均使用纯 C++ 手工实现（RLE 压缩、RC4 流密码、迭代密钥派生），无外部库依赖。
